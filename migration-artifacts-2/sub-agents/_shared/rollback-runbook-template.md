# Rollback Runbook Template

Every domain must produce a filled-in rollback runbook at `domains/{{DOMAIN}}/rollback-runbook.md` before its first canary ramp. Generated by `06-strangler-fig/canary-rollout.md`.

This template defines the structure; the canary-rollout agent fills it in.

---

```markdown
# Rollback Runbook — {{DOMAIN}}

**Owner**: {team or individual}
**On-call rotation**: {pagerduty schedule URL}
**Last drilled**: {date}

---

## Rollback Triggers (Automatic)

| Metric | Threshold | Window | Action |
|--------|-----------|--------|--------|
| Error rate (5xx) | > {X}% | 5 min | Auto-route 100% to legacy |
| p95 latency | > {Y} ms | 5 min | Auto-route 100% to legacy |
| Traffic delta (new vs legacy) | > {Z}% | 1 min | Alert + hold ramp |
| API diff rate | > 0.1% unexplained | continuous | Alert; manual decision |

---

## Rollback Procedures

### Level 1 — Flag flip (instant, no deploy)

```bash
# Via LaunchDarkly CLI (or your flag system)
ld-cli flag update {{DOMAIN}}.routing --variation legacy --comment "rollback at $(date -Iseconds)"
```

Expected effect: 100% of new requests go to legacy within {Y} seconds.

### Level 2 — Routing config revert (≤5 min)

If feature flags are bypassed or broken, revert the routing layer:

```bash
# nginx
git revert {commit-sha-of-routing-change}
sudo systemctl reload nginx

# Envoy via xDS
# (see domains/{{DOMAIN}}/strangler/routing-revert.yaml)

# Cloudflare Worker
wrangler rollback --name {{DOMAIN}}-router
```

### Level 3 — Code revert (≤30 min)

Revert the integration branch merge:

```bash
git checkout main
git revert -m 1 {merge-commit-sha}
git push origin main
# CI will redeploy legacy-only
```

### Level 4 — Data rollback (variable, 1h–24h)

Only required if dual-write window has produced divergent state.

1. Stop writes to new system (flag flip).
2. Snapshot current new-system DB.
3. Apply reconciliation script: `domains/{{DOMAIN}}/data-migration/reconcile.sql`.
4. Resume reads from legacy only.

See `domains/{{DOMAIN}}/data-migration.md` for full procedure.

---

## Communication

1. Post in `#incidents` channel: "Rolling back {{DOMAIN}} from {X}% to {legacy}. Trigger: {reason}."
2. Page tech lead if rollback level ≥ 3.
3. Open incident review ticket within 24h.

---

## Post-Rollback Checklist

- [ ] Confirm traffic is 100% on legacy via dashboard
- [ ] Confirm error rate dropped below threshold
- [ ] Capture: trigger, duration, MTTR, customer impact
- [ ] Update `migration-state.json.rollback_history`
- [ ] Schedule post-mortem within 5 business days
- [ ] Update `domains/{{DOMAIN}}/status` to `rolled_back`

---
