# Sub-Agent: Team Ownership Map

You map domains to owning teams/engineers and produce a `CODEOWNERS` template.

---

## Parameters

- `{{TEAM_ROSTER}}` (optional) — path to a YAML file listing teams, members, and areas of expertise. If absent, this agent emits placeholders and surfaces a human gate.

---

## Output Files

- `domains/_codeowners.md` — narrative mapping
- `.github/CODEOWNERS` (or update via PR if exists) — based on `templates/codeowners.txt`
- Update `migration-state.json.domains[].owner_team` and `.owner_engineers`

---

## Context Budget

Trivial. Reads only `domains/_index.md` and the roster.

---

## Algorithm

1. For each domain in `domains/_index.md`:
   - If roster has explicit assignment, use it.
   - Else use git log: top 2 contributors to files in this domain → suggested owners.
   - Else: mark `owner_team: TBD` and flag as blocker.

2. For shared-kernel libraries from `domains/_shared-kernel.md`: assign to a **platform team** or the domain that most heavily relies on it.

---

## Output: `domains/_codeowners.md`

```markdown
# Team Ownership Map

| Domain | Primary team | Lead engineer(s) | Backup team | Slack channel |
|--------|--------------|------------------|-------------|---------------|
| auth | team-iam | @alice @bob | team-platform | #iam |
| orders | team-commerce | @carol | team-payments | #commerce |
| payments | team-payments | @dave | team-commerce | #payments |
| ... |

## Shared kernel ownership

| Library | Owner | Reason |
|---------|-------|--------|
| lib/money | team-platform | used by orders, payments, billing |
| lib/auth-utils | team-iam | spawned from auth domain |

## Unassigned / TBD

- {domain} — needs owner. Action: tech-lead to assign by {date}.

## On-call rotation

Each domain owner team is responsible for canary supervision and rollback drills.
PagerDuty schedules:
- team-iam: {url}
- team-commerce: {url}
```
